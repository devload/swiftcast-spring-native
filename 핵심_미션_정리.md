## 💻 핵심 기술 요구사항

### 필수 조건
1. **크로스 플랫폼**: Windows + macOS 모두 지원
2. **런타임 독립**: JRE, Node.js 등 추가 설치 불필요
3. **단독 실행**: 더블클릭으로 즉시 실행

### 왜 Tauri + Rust?
```
Electron (X):
  - Node.js 런타임 필요
  - 파일 크기 ~100-200MB

Java (X):
  - JRE 설치 필요
  - 시작 시간 느림

Tauri (✅):
  - 런타임 불필요
  - 파일 크기 ~5-10MB
  - 즉시 시작
  - Windows + macOS 네이티브
```

## 💡 핵심 가치 (우선순위 순)

### 1️⃣ **유연성** (가장 중요!)
- 상황에 따라 Provider 선택
- Claude 필요할 때: Anthropic
- 비용 절감 필요할 때: GLM
- 한 번의 클릭으로 전환

### 2️⃣ **투명성**
- Provider별 사용량 명확히 파악
- 토큰, 비용, 시간 추적
- "이번 주 Claude 얼마, GLM 얼마 썼는지" 확인

### 3️⃣ **사용 편의성**
- Desktop UI로 관리
- 복잡한 설정 없이

### 4️⃣ **비용 최적화**
- 선택의 결과로 얻는 효과
- 필요할 때만 비싼 API

## 🚀 실제 사용 시나리오

### 시나리오 A: 평소 Claude, 간단한 작업 GLM
```
1. SwiftCast에 Anthropic, GLM 계정 모두 등록
2. 기본: Anthropic 활성화
3. 간단한 작업할 때: GLM으로 전환
4. 대시보드에서 각 Provider별 사용량 확인
```

### 시나리오 B: GLM 주력, Claude 백업
```
1. GLM을 주 계정으로
2. Claude Code 작업
3. GLM 한도 도달 시 Anthropic으로 전환
4. 통계로 총 비용 확인
```
